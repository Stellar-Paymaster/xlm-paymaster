# Consolidated Performance & Benchmark Reports

This document serves as the canonical index for all performance and benchmark
reports generated by the XLM Paymaster platform.  Individual report files live in this
`docs/reports/` directory rather than scattered across sub-project roots.

## Available Reports

| File | Description |
|------|-------------|
| [migration_benchmark_report.md](./migration_benchmark_report.md) | Database migration timing over a 100k-transaction dataset |
| [tokio_performance_report.md](./tokio_performance_report.md) | Tokio async-runtime RPS benchmark across multiple thread configs |
| [load_testing_report.md](./load_testing_report.md) | Peak load testing results targeting 1,000 RPS using k6 and Locust |

## Summary

### Load Testing (k6 & Locust)

- Target: 1,000 RPS against `/fee-bump`
- Bypass configuration: `FLUID_DISABLE_RATE_LIMITS=true`
- Achieved throughput: **~1,012 - 1,024 RPS** with **0.00% error rate**
- Peak CPU utilization: **~42%** on a 4-vCPU staging VM

### Migration Benchmark

- Migration `20260329180000_add_tenant_region` applied successfully
- Dataset: 10,000 tenants / 100,000 transactions
- Execution time: **3524 ms**, memory overhead: **0.07 MB**

### Tokio Runtime Performance

- Best config: `high_concurrency` (20 worker threads, 80 max-blocking)
- Peak throughput: **18,110 RPS** at **0.06 ms** avg latency
- All configurations achieved **100 % success rate**

## Generating New Reports

```bash
# Migration benchmark (from server/)
cd server && pnpm run benchmark:migration

# Tokio performance (from paymaster-server/)
cd paymaster-server && cargo bench
```

Move any generated `.md` reports into `docs/reports/` and update the table
above.
